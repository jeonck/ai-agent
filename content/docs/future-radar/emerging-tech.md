## 1. 음성·실시간 에이전트

**현황**: 음성 인터페이스와 에이전트의 결합이 빠르게 성숙 중
**주요 변화**: 텍스트 기반 에이전트가 실시간 음성 대화로 확장

**준비할 것**
- 음성 → 텍스트 → 에이전트 → 텍스트 → 음성 파이프라인 이해
- 지연 시간 최적화 (음성 응답은 3초 내가 기준)
- 음성 전용 가드레일 (발화 내용의 PII 처리)

## 2. 멀티모달 에이전트

**현황**: 이미지·문서·음성을 함께 처리하는 에이전트가 실용화 단계
**주요 변화**: 시각 정보를 이해하고 행동하는 에이전트 (스크린샷, 차트, 문서 이미지)

**실용 사례**
- 청구서/계약서 이미지 → 데이터 추출
- 대시보드 스크린샷 → 인사이트 생성
- 공장 카메라 피드 → 이상 탐지

## 3. 장기 실행형 에이전트

**현황**: 수분-수시간이 아닌 수일-수주 동안 실행되는 에이전트 등장
**주의사항**: 상태 관리, 오류 복구, 비용 통제가 핵심 과제

**기술적 도전**
- 장기 메모리 일관성 유지
- 중간 실패 시 재개(resume) 전략
- 비용이 무한정 증가하지 않도록 상한 설정

## 4. 에이전트 간 상호운용성 (A2A)

**현황**: A2A 프로토콜이 Linux Foundation AAIF에서 표준화 진행 중
**의미**: 서로 다른 회사의 에이전트가 표준 프로토콜로 협력 가능

**실무 준비**
- 에이전트 카드(Agent Card) 개념 이해
- MCP(도구 연결)와 A2A(에이전트 조율) 명확히 구분
- 외부 에이전트 신뢰도 검증 방법 마련

## 5. 컴퓨터 사용형 에이전트 (Computer Use)

**현황**: 브라우저·데스크톱을 직접 조작하는 에이전트가 기업 환경에 진입
**주의사항**: 보안 위험이 매우 높음 — 격리된 환경에서만 사용

**보안 필수 사항**
- 샌드박스 환경에서만 실행
- 네트워크 접근 최소화
- 실행 내용 전체 녹화·감사
