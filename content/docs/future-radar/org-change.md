## AI 운영 조직 (AI Ops) 설계

에이전트가 프로덕션에 들어가면 새로운 운영 역할이 필요합니다.

**새로 필요한 역할**

| 역할 | 주요 책임 |
|------|---------|
| **AI 엔지니어** | 에이전트 설계·구현·최적화 |
| **AI 운영 엔지니어** | 프로덕션 모니터링·장애 대응 |
| **Prompt 엔지니어** | 프롬프트 설계·eval 관리 |
| **AI 거버넌스 담당** | 정책·감사·컴플라이언스 |

**기존 역할의 변화**

- **소프트웨어 엔지니어**: 비결정론적 컴포넌트 설계 역량 추가 필요
- **데이터 엔지니어**: RAG 파이프라인·벡터 DB 운영 역량 추가
- **보안팀**: 프롬프트 인젝션·에이전트 공격면 이해 필요

## 사람-에이전트 협업 모델

**세 가지 협업 패턴**

### 패턴 1: 에이전트가 초안, 사람이 결정
- 에이전트: 분석·초안·옵션 생성
- 사람: 최종 선택·승인
- 적합: 고위험 결정, 창의적 작업

### 패턴 2: 사람이 감독, 에이전트가 실행
- 에이전트: 자율 실행
- 사람: 이상 감지 시 개입
- 적합: 반복적이고 규칙이 명확한 업무

### 패턴 3: 에이전트가 완전 자율 실행
- 에이전트: 계획부터 실행까지
- 사람: 사후 감사·성과 검토
- 적합: 저위험·고반복 업무 (충분한 eval 후)

## 직무 재설계 원칙

{{< callout type="info" >}}
에이전트가 일자리를 "없애는" 것이 아니라 **어떤 일을 하는지를 바꿉니다**. 반복·처리 업무 → 판단·감독·예외 처리로 이동합니다.
{{< /callout >}}

**재설계 방향**
- 에이전트가 처리하는 업무: 반복적, 규칙 기반, 대량 처리
- 사람이 집중하는 업무: 예외 처리, 복잡한 판단, 고객 관계, 전략
