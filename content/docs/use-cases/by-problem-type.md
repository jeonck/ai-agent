## 1. 문서 해석형

**정의**: 비정형 문서를 읽고 구조화된 정보를 추출하거나 판단을 내리는 업무

**적합 조건**
- 입력이 텍스트, PDF, 이메일, HTML 등 비정형
- 판단 기준이 복잡하고 문맥에 따라 달라짐
- 처리량이 많아 사람이 일일이 검토하기 어려움

**예시**
- 계약서 리뷰 (위험 조항 탐지)
- 규정 준수 문서 검토
- 고객 피드백 분류·요약
- 특허 문서 분석

## 2. 멀티스텝 업무처리형

**정의**: 여러 단계가 순차·병렬로 이어지며, 각 단계 결과가 다음 단계 입력이 되는 업무

**적합 조건**
- 5단계 이상의 처리 과정
- 단계 간 상태(컨텍스트)를 유지해야 함
- 각 단계마다 다른 시스템·도구를 사용

**예시**
- 구매 요청 → 벤더 조회 → 견적 비교 → 승인 요청 → 발주
- 이슈 접수 → 분류 → 담당자 배정 → 처리 → 결과 통보

## 3. 승인·검수형

**정의**: 사람의 최종 승인이 필요하지만, 그 전 단계를 자동화하는 업무

**Human-in-the-Loop 설계 원칙**
- 에이전트가 초안 작성 + 근거 제시
- 사람이 승인/거부/수정
- 이력 기록 + 감사 추적

**예시**
- 콘텐츠 발행 전 검수
- 예외 처리 요청 검토
- 고위험 API 호출 전 승인

## 4. 시스템 오케스트레이션형

**정의**: 여러 외부 시스템을 조율하여 복합 목표를 달성하는 업무

**적합 조건**
- 3개 이상 시스템 연동
- 시스템 간 오류 처리·재시도 필요
- 트랜잭션 일관성 보장 필요
