## 에이전트를 써야 하는 업무 체크리스트

아래 항목 중 **3개 이상** 해당하면 에이전트가 유효한 선택입니다.

- [ ] 업무 단계가 5개 이상이고, 각 단계가 이전 결과에 의존한다
- [ ] 비정형 텍스트(이메일, 문서, 대화)를 판단 근거로 사용한다
- [ ] 여러 시스템(DB, API, 파일)에 순차적으로 접근해야 한다
- [ ] 예외 상황마다 다른 처리 경로가 필요하다
- [ ] 규칙으로 모든 케이스를 커버하기 어렵다
- [ ] 사람이 중간 판단을 내려야 하는 포인트가 있다

## 올바른 도구 선택 의사결정 트리

```
질문: 이 업무를 자동화하고 싶다
         │
         ▼
규칙 기반으로 100% 커버 가능? ──YES──▶ RPA / 코드 자동화
         │
         NO
         ▼
단순 Q&A / FAQ 응답? ──────────YES──▶ RAG + 챗봇
         │
         NO
         ▼
문서 검색·요약이 핵심? ────────YES──▶ RAG 에이전트
         │
         NO
         ▼
멀티스텝 + 도구 사용 필요? ────YES──▶ 단일 에이전트 + MCP
         │
         NO
         ▼
여러 전문가 역할 필요? ─────────YES──▶ 멀티에이전트
```

## RAG로 충분한가, 에이전트가 필요한가

| 기준 | RAG로 충분 | 에이전트 필요 |
|------|-----------|--------------|
| 업무 성격 | 정보 검색·요약 | 정보 검색 + 행동 실행 |
| 도구 사용 | 없음 | API·DB·파일 시스템 |
| 상태 유지 | 불필요 | 멀티턴·장기 실행 |
| 오류 처리 | 재시도 없음 | 자가 수정 필요 |
| 복잡도 | 낮음 | 중~높음 |

{{< callout type="warning" >}}
**과설계 경고**: 많은 팀이 RAG로 충분한 문제에 에이전트를 도입합니다. 에이전트는 복잡도·비용·장애 가능성을 높입니다. 단순한 해법이 먼저입니다.
{{< /callout >}}

## PoC 후보 선정 기준

좋은 첫 번째 PoC 업무의 조건:

1. **임팩트** — 자동화 시 시간/비용 절감이 측정 가능
2. **경계 명확** — 입력과 출력이 정의 가능
3. **실패 허용** — 잘못 처리해도 복구 가능
4. **데이터 접근** — 필요한 데이터에 접근할 수 있음
5. **소규모** — 2~4주 내 결과 확인 가능
