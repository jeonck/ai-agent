## 2026년 현재 에이전트 생태계

2026년 에이전트 생태계의 무게중심은 **"어떻게 만드느냐"에서 "어떻게 안정적·안전하게 운영하느냐"**로 이동했습니다.

## 프로토콜 계층 (통신 표준)

| 프로토콜 | 역할 | 상태 |
|---------|------|------|
| **MCP** (Model Context Protocol) | 에이전트 ↔ 도구 연결 | Linux Foundation AAIF 표준화 |
| **A2A** (Agent-to-Agent) | 에이전트 ↔ 에이전트 통신 | ACP 병합, AAIF 거버넌스 |

{{< callout type="info" >}}
**핵심 구분**: MCP는 도구를 노출하는 프로토콜이고, A2A는 에이전트를 조율하는 프로토콜입니다. 이 경계를 명확히 해야 아키텍처 설계가 흔들리지 않습니다.
{{< /callout >}}

## 프레임워크 계층 (개발 도구)

| 프레임워크 | 주요 특징 | 상태 |
|-----------|----------|------|
| LangGraph | 상태 그래프 기반 워크플로우 | 활발 유지보수 |
| CrewAI | 역할 기반 멀티에이전트 | 활발 유지보수 |
| Google ADK | Google 생태계 통합 | GA |
| Microsoft Agent Framework 1.0 | 엔터프라이즈 오케스트레이션 | 2026.04 GA |
| AutoGen | 대화형 멀티에이전트 | 유지보수 모드 |

{{< callout type="warning" >}}
**프레임워크는 변동성이 큽니다.** 6개월 단위로 지형이 바뀝니다. 프레임워크 선택보다 프로토콜 표준(MCP/A2A)을 기반으로 설계하는 것이 장기적으로 안전합니다.
{{< /callout >}}

## 아직 안 풀린 문제

프로덕션에서 모든 팀이 직접 해결해야 하는 미해결 과제:

- **관측성**: 어떤 프로토콜도 구조화 로깅·OpenTelemetry 통합을 강제하지 않음
- **에이전트 신원**: W3C DID·검증가능 자격증명 기반 에이전트 인증은 로드맵 단계
- **비용 예측**: 장기 실행 에이전트의 토큰 비용 추정 어려움
- **평가 표준**: 에이전트 품질 측정 방법론 미표준화

이 "아직 안 풀린 문제"를 이해하는 것이 플랫폼 엔지니어 관점의 핵심 경쟁력입니다.
