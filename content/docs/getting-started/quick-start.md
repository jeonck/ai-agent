역할에 따라 가장 효율적인 학습 경로가 다릅니다.

## 리더용 30분 이해 코스

**목표**: AI 에이전트 도입 결정을 위한 비즈니스 이해

1. [에이전트란 무엇인가](/docs/getting-started/what-is-agent/) — 챗봇·워크플로우·에이전트 구분 (10분)
2. [도입 판단](/docs/getting-started/decision/) — 우리 조직에 맞는가? 체크리스트 (10분)
3. [유즈케이스: 부서별](/docs/use-cases/by-department/) — 우리 업무에 적용할 수 있는가? (10분)

**이후 읽으면 좋은 것**: 거버넌스, 산업별 플레이북

## 실무 기획자용 60분 적용 코스

**목표**: 첫 번째 에이전트 PoC 기획

1. [에이전트란 무엇인가](/docs/getting-started/what-is-agent/) (10분)
2. [도입 판단](/docs/getting-started/decision/) — 도입 체크리스트 + 의사결정 트리 (10분)
3. [우선순위 판단 도구](/docs/use-cases/priority-tools/) — ROI 매트릭스 + PoC 캔버스 (15분)
4. [패턴 선택 가이드](/docs/design-patterns/pattern-selection/) — 어떤 아키텍처인가? (15분)
5. [고객지원 플레이북](/docs/industry-playbooks/customer-support/) — 참조 구현 예시 (10분)

**이후 읽으면 좋은 것**: 평가(Eval), 가드레일 설계

## 개발자용 90분 빌드 코스

**목표**: 단일 에이전트 구현 착수

1. [단일 에이전트 패턴](/docs/design-patterns/single-agent/) — 패턴 이해 (15분)
2. [핵심 컴포넌트](/docs/implementation/core-components/) — 모델·도구·지침 (20분)
3. [컨텍스트 엔지니어링](/docs/implementation/context-engineering/) — 성능 핵심 (20분)
4. [시스템 연결 (MCP)](/docs/implementation/system-connection/) — 도구 연결 방법 (15분)
5. [에이전트 평가 (Eval)](/docs/ops-eval-governance/evaluation/) — 테스트 설계 (20분)

**이후 읽으면 좋은 것**: 워크플로우 패턴, 관측성, 가드레일

## 보안·운영팀용 60분 거버넌스 코스

**목표**: 에이전트 도입 리스크 이해와 통제 방안

1. [2026 기술 지형도](/docs/getting-started/landscape-2026/) — 현재 생태계 이해 (10분)
2. [가드레일 설계](/docs/ops-eval-governance/guardrails/) — 보안 통제 계층 (20분)
3. [운영: 관측성·비용·장애](/docs/ops-eval-governance/operations/) — 런타임 통제 (15분)
4. [거버넌스](/docs/ops-eval-governance/governance/) — NIST AI RMF·정책 설계 (15분)

**이후 읽으면 좋은 것**: 미래 레이더 (컴퓨터 사용형 에이전트 보안)
